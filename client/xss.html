Cross-Site Scripting (XSS) is a security vulnerability that allows attackers to inject malicious scripts into web pages viewed by other users. These scripts can steal information, manipulate web content, or perform actions on behalf of the user.

How to Prevent XSS:
Sanitize Input: Always sanitize user input to remove or escape harmful characters.
Use Output Encoding: Encode data before displaying it in the browser.
Content Security Policy (CSP): Implement CSP to restrict the sources from which scripts can be loaded.

Relation to C: In C programming, functions like gets() are insecure because they do not validate input length. Always validate inputs to prevent vulnerabilities.
